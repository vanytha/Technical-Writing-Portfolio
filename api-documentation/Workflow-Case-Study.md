# How Payment Authorization Works in a SaaS Platform

This document explains the end-to-end flow of a **payment authorization** in a typical SaaS application — from the moment a user submits card details to the point where the system logs the result.

### Payment Authorization Flow

1. The user enters card details in the checkout interface.
2. The backend server sends a payment authorization request to the Moneris API.
3. Moneris communicates with the card network and issuing bank.
4. The payment gateway returns the transaction result.
5. The SaaS platform updates internal records and notifies the user.
   
## 1) User Input

The user enters payment details in the SaaS checkout UI (web or mobile).

**Typical inputs:**
- Card number, expiry date, CVV
- Cardholder name
- Billing address (optional, but common for fraud checks)
- Amount and currency
- Email / user ID (for receipts and account mapping)

## 2) Payment Request to Moneris API

The backend server formats the payment details and sends them to the **Moneris Payments API** for processing.

### Typical inputs:
- Order ID  
- Total amount  
- Currency code  
- Customer ID  
- Payment method  
- Card details (number, expiry date, CVV)  
- Cardholder name  
- Billing address  
- Customer email  
- Transaction type  

---

## 3)  Payment Gateway Response (Moneris)

After the payment request is processed, the backend receives a response from the Moneris Payments API.  
The response contains details about the transaction status and identifiers generated by the payment gateway.

---

## Typical Response Fields

| Field | Description |
|------|-------------|
| Payment Status | Indicates whether the transaction was approved, declined, or pending |
| Transaction Number | Unique identifier assigned by the payment gateway for the transaction |
| Authorization Number | Code issued by the issuing bank confirming approval |
| Credit Card Type | Type of card used for the payment (Visa, MasterCard, etc.) |
| Third-Party Processor | Payment processor or network involved in the transaction |

---

## Example Payment Response

| Field | Example Value |
|------|---------------|
| Payment Status | Approved |
| Transaction Number | txn_789456123 |
| Authorization Number | auth_456789 |
| Credit Card Type | Visa |
| Third-Party Processor | Moneris |

## 4) Handle Successful Payment

When the payment is successfully processed by the payment gateway, the system performs the following actions.

---

## Typical Actions

| Action | Description |
|------|-------------|
| Display Success Message | Show a confirmation message in the UI indicating that the payment was successful |
| Update Payment Records | Update the payment status and transaction details in the `internalorderpayments` table |
| Store Transaction Details | Save transaction number, authorization number, and card type for future reference |
| Send Confirmation Email | Send a payment confirmation email or receipt to the user |

---

## Example Scenario

| Step | Example |
|-----|---------|
| UI Message | "Payment successful. Your order has been confirmed." |
| Database Update | `internalorderpayments.status = Success` |
| Stored Transaction Number | `txn_789456123` |
| Authorization Number | `auth_456789` |
| Email Notification | Confirmation email sent to `john.doe@email.com` |

## 5) Handle Payment Errors and Update Records

If the payment fails or an issue occurs during processing, the system performs the following actions.

---

## Typical Actions

| Action | Description |
|------|-------------|
| Display Error Message | Show a generic payment error message in the UI to inform the user that the transaction could not be completed |
| Update Payment Records | Update the payment status and related details in the `internalorderpayments` table |
| Send Email Notification | Send an email to the user informing them about the payment status (success or failure) |
| Log Transaction Details | Store error codes or response details for troubleshooting and audit purposes |

---

## Example Scenario

| Step | Example |
|-----|---------|
| UI Message | "Payment could not be processed. Please try again or use a different payment method." |
| Database Update | `internalorderpayments.status = Failed` |
| Email Notification | Email sent to `john.doe@email.com` informing that the payment attempt was unsuccessful |
| Logged Error | `PAYMENT_DECLINED` returned from the payment gateway |
